按照既定的七步详细拆解要求，对这道“乘积最大子数组”的动态规划解法进行全方位解析：

### （1）题目白话文解释

题目的核心诉求是：给定一排整数（也就是数组），要求从中切出一段**连续不断**的片段，使得这个片段里所有数字相乘的**乘积结果尽可能大**。如果切出来的片段只有一个数字，那乘积就是这个数字本身。必须至少挑选一个数字。

**生僻/关键概念解释：**
* **连续子数组**：强调“连续”二字。比如数组是 `[1, 2, 3, 4]`，那么 `[2, 3]` 是连续子数组，但 `[2, 4]` 不是，因为中间跳过了 `3`。
* **乘积的负负得正特性**：与求“最大子数组和”不同，乘法有一个极其特殊的性质：两个极小的负数相乘，会瞬间变成一个极大的正数。这是解开本题的最核心难点。

---

### （2）代码解题思路来源

这段代码采用的是**动态规划（Dynamic Programming）**算法，并且进行了**空间优化**（将原本需要一维数组保存的状态压缩到了几个简单变量中）。

解题思路从乘法的特殊性出发：
1.  **为何不能只记录最大值？** 如果只记录“到当前数字为止的最大乘积”，当遇到一个负数时，原本最大的正数乘积瞬间变成了极小的负数；但如果紧接着下一个数字依然是负数，那个极小的负数乘积又会瞬间反转成极大的正数。如果之前扔掉了那个负数乘积，就无法捕捉到反转后的最大值。
2.  **双状态维护**：为了应对“负负得正”，在遍历数组时，必须**同时记录两个极值**：
    * 以当前数字结尾的**最大乘积**（`f_max`）。
    * 以当前数字结尾的**最小乘积**（`f_min`）（这个最小值往往是个绝对值很大的负数，是潜在的“翻盘”种子）。
3.  **状态转移策略**：当遍历到一个新数字 `x` 时，要想构成一段以 `x` 结尾的连续子数组，有三种选择：
    * 选择一：让 `x` 跟随之前的最大乘积序列，即 `f_max * x`。
    * 选择二：让 `x` 跟随之前的最小乘积序列（如果 `x` 是负数，最小乘积也是负数，这一步就会诞生巨大的正数），即 `f_min * x`。
    * 选择三：前面的序列统统不要，因为前面可能是个绝对值很小的数甚至是0，不如 `x` 另起炉灶单干，即只有 `x` 本身。
4.  将以上三种选择的结果进行比较，最大的挑出来作为新的 `f_max`，最小的挑出来作为新的 `f_min`。同时不断更新历史最高记录 `ans` 即可。

### （3）带有注释的代码

```python
class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        # 初始化全局最大答案，设为负无穷大，确保能被数组里的任何真实乘积覆盖
        ans = -inf  
        # 初始化当前结尾的最大乘积和最小乘积。
        # 设为 1 的原因是 1 是乘法单位元，任何数乘 1 还是本身，便于第一步的推导
        f_max = f_min = 1
        # 依次遍历数组中的每一个数字 x
        for x in nums:
            # 核心状态转移方程：
            # 在“延续最大值”、“延续最小值（期待负负得正）”和“另起炉灶单干”三者中，
            # 找出新的局部最大值和局部最小值。
            # 注意：必须写在同一行并行赋值，防止新的 f_max 污染 f_min 的计算
            f_max, f_min = max(f_max * x, f_min * x, x), min(f_max * x, f_min * x, x)
            # 用刚刚算出的局部最大乘积 f_max 来挑战并更新全局历史最大乘积 ans
            ans = max(ans, f_max)
        # 遍历完毕，返回历史最高记录
        return ans
```

### （4）每一行代码详细解释

1.  `class Solution:`：定义执行类，符合 LeetCode 提交标准规范。
2.  `def maxProduct(self, nums: List[int]) -> int:`：定义核心执行方法，接收整数列表 `nums`，返回最大乘积，类型为整数 `int`。
3.  `ans = -inf`：定义用于存放最终答案的变量 `ans`。因为数组中可能全都是负数，最终答案也可能为负，所以将其初始化为 Python 中的负无穷大（`-inf`），确保第一次更新时必定能被真实数值替换。
4.  `f_max = f_min = 1`：定义并初始化动态规划的两个核心状态变量。`f_max` 记录“包含当前数字的子数组最大乘积”，`f_min` 记录“包含当前数字的子数组最小乘积”。初始值设为 `1`，使得后续遇到第一个元素 `x` 时，`1 * x` 正好等于 `x` 本身，完成启动。
5.  `for x in nums:`：启动循环，将列表 `nums` 中的数字逐一提取，赋值给当前变量 `x`。
6.  `f_max, f_min = max(f_max * x, f_min * x, x), min(f_max * x, f_min * x, x)`：这是全段代码中最核心的一行。利用 Python 的元组解包特性进行**并行赋值**。同时计算三个候选值：基于先前最大乘积的延伸（`f_max * x`）、基于先前最小乘积的延伸（`f_min * x`）、以及斩断过去仅保留当前元素（`x`）。从中分别挑出最大值赋予新的 `f_max`，挑出最小值赋予新的 `f_min`。并行赋值保证了等号右侧使用的都是上一轮的旧值，不会发生数据覆盖干扰。
7.  `ans = max(ans, f_max)`：每结束一轮数字处理，就拿刚刚得出的以当前数字结尾的局部最大乘积 `f_max`，去和历史最高记录 `ans` 比较。如果更大，就刷新最高记录。
8.  `return ans`：所有数字遍历结束，`ans` 固化为整个数组中乘积最大的子数组结果，将其输出返回。
